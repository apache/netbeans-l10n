调试器命令行

参数:
    platformhome - 在其中填写平台的起始目录
    device - 在其中填写设备名称
    classpath - 将在下一个向导步骤中定义的平台类路径
    jadfile - jad 文件名
    jadurl - 用于 OTA 执行的 jad url
    securitydomain - 用于 OTA 执行的安全域
    debug - 这只是一个表示调试模式的条件
    debugaddress - 调试地址
    debugtransport - 调试传输
    debugserver - 调试服务器模式 - 默认设置为 true
    debugsuspend - 调试挂起模式 - 默认设置为 true
    cmdoptions - 来自项目设置的命令行选项
    / - 特定于 OS 的目录分隔符

参数格式:
{argument} 是一个简单的替换变量, 例如, {jadfile} 将替换为实际的 jad 文件路径。
{argument|command line part} 是一个条件, 如果将 argument 设置为某个值, 则 command line part 将会添加到命令行中。
允许将条件和简单的替换变量结合在一起使用。例如: {jadfile|-Xdescriptor:"{jadfile}"} 表示如果 jadfile 参数包含 /myfolder/my.jad, 则 -Xdescriptor:"/myfolder/my.jad" 将会添加到命令行。
